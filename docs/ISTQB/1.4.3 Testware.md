### 1.4.3. Testware

चला, आता आपण **1.4.3. Testware** कडे वळूया.

'Testware' म्हणजे सोप्या भाषेत सांगायचं तर **"टेस्टिंग करताना आपण जे काही डॉक्युमेंट्स किंवा साहित्य तयार करतो, ते सर्व."** जसं स्वयंपाक करताना लागणारी भांडी, साहित्य आणि रेसिपी बुक असतं, तसंच हे आहे.

परीक्षेच्या दृष्टीने कोणतं 'Testware' कोणत्या 'Activity' मध्ये येतं, हे लक्षात ठेवणं गरजेचं आहे:

### **१. Planning (नियोजन)**

- **Work Products:** Test Plan, Test Schedule, Risk Register.
- **महत्त्वाचं:** 'Entry Criteria' (कधी सुरू करायचं) आणि 'Exit Criteria' (कधी थांबायचं) हे सुद्धा प्लॅनचाच भाग असतात.

### **२. Monitoring & Control (देखरेख)**

- **Work Products:** Test Progress Reports (आपण किती मागे/पुढे आहोत).
- **eg.:** 'Daily Stand-up' मध्ये आपण काय काय पूर्ण केलं, काय अडचण आली, याची नोंद ठेवणं. aani Progress Report tayar karne. (Reports, Charts, Status)

### **३. Analysis & Design (काय आणि कसं?)**

- **Analysis:** Test Conditions (काय टेस्ट करायचंय).
- **Design:** Test Cases, Test Data Requirements, Test Environment Requirements.

### **४. Implementation (तयारी)**

- **Work Products:** Test Procedures, **Test Scripts** (Manual/Automated), Test Suites, Test Data.
- **विशेष:** इथे 'Stubs' आणि 'Drivers' सारखे तांत्रिक घटक सुद्धा तयार केले जातात.
- eg. (Scripts, Data, Procedures)

### **५. Execution (प्रत्यक्ष रन)**

- **Work Products:** Test Logs (काय घडलं त्याची नोंद) आणि **Defect Reports** (सापडलेल्या चुका).

### **६. Completion (सांगता)**

- **Work Products:** Test Completion Report आणि **Lessons Learned** (पुढच्या वेळी काय चांगलं करता येईल).

**A)** Test Implementation work product

**C)** Test Completion work product
